Welcome dungeon masters, adventurers, and monster enthusiasts!

This is the Dungeons & Dragons 5th Edition Monster Database! This database is designed for any dungeon master (DM) or player looking for more information about the monsters that they can possibly encounter throughout their campaign. The data for this web app came from the "DnD 5e Monsters" dataset by Brandon Patterson on Kaggle.com. The "DnD 5e Monsters" dataset was created by scraping data from AideDD.org, an online resource for D&D.

What does the data look like?

This web app utilizes a relational database. Users can play around with the main table in the database on the Dataset page. This table includes information on 762 unique monsters: each row represents a single creature, and each column represents a specific characteristic. The data in this table is slightly different from the original, as the values in some columns have been re-formatted or split into multiple attributes. The primary key for the table would be the variable "name", as its the simplest column that uniquely describes each row. The other columns in the dataset are as follows:

Attributes
  • Type: describes the fundamental nature of a monster. A monster's type can have special interactions with certain spells, magic items, class features, and other effects.
  • Subtype: describes the monster's type in more detail. Not all monsters have a subtype. Also known as a tag.
  • Swim: an indicator variable that equals 1 if a monster can swim and 0 otherwise.
  • Fly: an indicator variable that equals 1 if a monster can fly and 0 otherwise.
Encounter Type
  • Challenge rating (CR): a quantitative measure of how difficult a monster is. As a rule of thumb, 4 players should be able to take down a monster with a CR equal to their overall level without any party casualities. CR determines how much XP a monster gives upon its defeat.
  • Size: a measure of how much space a monster occupies. Size affects how a monster can be interacted with and interacts with the environment around it. Size ranges from Tiny, Small, Medium, Large, Huge, or Gargantuan.
  • Alignment: suggests how a monster might interact with other creatures and player characters. Alignment has two axes: lawful to chaotic, and good to evil. Not all monsters have a defined alignment and can be determined by the DM.
  • Legendary: an indicator variable that equals 1 if a monster is legendary and 0 otherwise. Legendary creatures have unique abilities and effects that ordinary monsters do not.
Combat Stats
  • Armor class (AC): how difficult it is to deal damage to a monster. A monster with a higher AC is more difficult to hit.
  • Hit points (HP): how much damage a monster can sustain until defeated. Players can deal lethal or non-lethal damage to a monster until its HP reaches 0.
  • Strength (Str): a measure of a monster's physical ability.
  • Dexterity (Dex): a measure of a monster's proficiency with speed, agility, and finesse.
  • Constitution (Con): a measure of a monster's fortitude and hardiness.
  • Intelligence (Int): a measure of a monster's book smarts.
  • Wisdom (Wis): a measure of a monster's street smarts.
  • Charisma (Cha): a measure of a monster's personability.
References
  • URL: the link for the monster's AideDD page, if available.
  • Source: the book where the monster's stats are located.
